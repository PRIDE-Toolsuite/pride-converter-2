File Selection

Previous | Next | See Also

This screen allows you to select the input file(s) that you wish to convert into PRIDE XML and set any parser-specific options, if available. You may also select FASTA-formatted protein sequence files and mzTab files to use as additional annotations.

Source File

The sourcefile is the search engine's ouput file. You must provide at least one source file before you can proceed.

FASTA File (optional)

PRIDE Converter can include the searched protein sequences in the generated PRIDE XML files. This is recommended when non-standard protein databases are used for the search. If you use this option you should provide the exact same fasta file as was used for the search.

There are multiple formats for the header in a FASTA file. When providing the FASTA file you must select the appropriate method to extract the protein accession from the FASTA file header. The method depends on the used search engine as well as the type of FASTA file.

mzTab File (optional)

mzTab files can be provided to include quantitative and or gel-related information into the generated PRIDE XML file. Skeleton mzTab files can be generated using the "PRIDE mzTab Generator". The same options must be used when generating the mzTab file as are used during the conversion.

Detailed information on how to add quantitative and or gel-related data into skeleton mzTab files can be found in the PRIDE Converter wiki at http://code.google.com/p/pride-converter-2/wiki/PrideConverter2.

Spectrum File (required if available)

Some search engine output files do not contain the searched spectra but only the identifications. For these search engines the searched MS data file must be provided in the "Spectrum File" option.

Parser Options

Some formats allow parser-specific options to be set. These, if any, will be displayed in the "Parser Options" table. Documentation on the meaning of the various options can be obtained by clicking on the "Explain Options" button. To edit the value of an option, you need to double-click on it and then you will be able to modify its value.

Advanced options

By design, PRIDE Converter keeps intermediate work files called report files in the same directory as the source file(s). If such a report file is already present, PRIDE Converter will try to reload it so that the conversion process can be continued from where it was previously stopped. If report files are not present or are invalid, they will be created. You can force PRIDE Converter to ignore previous report files by clicking on the "Force generation of new report file" checbox.

During conversion PRIDE Converter tries to correct malformatted protein accessions. If separate target and decoy databases were used the option "Search database contains decoy sequences" should be disabled.

Notes

By default, users will be shown a form that will only provide single-file selection dialogues but users who wish to do multiple file batch conversion can switch to an alternative view, which will allow them to select multiple files of each type (source, mzTab, spectra, etc).

When converting multiple files at once, a warning message will be displayed to the user indicating that some metadata will be copied across all files while some will not. The information that will be common across all files will include: sample annotations, post-translational modifications (PTM), protocol and instrumentation description and any other experiment-level annotations. Software processing, which can include file-specific software settings, score thresholds and false discovery rates will be taken from the individual report files associated with each input file to be converted.

Once the files have been selected, the PRIDE Converter 2 runs in prescan mode to generate the report files, where the annotations that the user will input in the coming steps will be stored prior to actual conversion. Report file generation can take anywhere between seconds and several minutes and is directly proportional to the size and complexity of the input files. If several files are being converted at once, a report file needs to be generated for each input file, which will compound the time required.
